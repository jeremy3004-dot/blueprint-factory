# Codex + Blueprint Factory Video Playlist

Research date: 2026-06-29

Use this for onboarding new Blueprint Factory builders. The goal is not to make them watch every AI coding video on YouTube. The goal is to give them enough taste, workflow discipline, and frontend craft to work inside this repo without wandering.

## How To Use This Playlist

Give new builders the core playlist first. It is about one long day of watching, or two half-days with hands-on practice.

After each block, they should do one small factory exercise:

- Ask Codex to explain this repo and summarize the factory rules.
- Run a Blueprint Factory status check on an existing site.
- Find a strong donor site and write a donor topology.
- Make one small visual improvement and capture before/after screenshots.

They should read these local docs alongside the videos:

- `AGENTS.md`
- `README.md`
- `factory/playbooks/blueprint-factory-callbook.md`
- `factory/playbooks/reference-first-build.md`
- `factory/playbooks/green-pastures-pack.md`
- `factory/qa/beauty-pass-rubric.md`

## Blueprint Factory Training Prompt

After the playlist, give each new builder this prompt inside the repo:

```text
Read AGENTS.md, README.md, factory/playbooks/blueprint-factory-callbook.md,
factory/playbooks/reference-first-build.md, and factory/qa/beauty-pass-rubric.md.

Then explain the Blueprint Factory workflow to me as a checklist:
1. How to start a normal high-craft website.
2. How donor-first cloning works.
3. What files every site must have.
4. What proof is required before a site is ready.
5. What you must ask before deploying or making major art-direction changes.
```

Then give them this practical exercise:

```text
Pick one existing site under sites/.
Inspect the current factory state.
Find the next missing gate.
Do not edit yet.
Tell me the smallest useful next action and what proof you would collect.
```

## What Good Looks Like

A trained builder should be able to say:

- Codex needs clear task boundaries, repo context, and verification.
- Blueprint Factory starts with a visual donor, not blank-page invention.
- The first pass clones structure and craft; the second pass translates brand, copy, assets, and goals.
- Every site needs brief, art direction, asset log, deploy notes, screenshots, and visual review.
- Animation matters only when it supports the signature moment.
- A task is not done until the build, screenshots, motion evidence, and Beauty Pass prove it.
