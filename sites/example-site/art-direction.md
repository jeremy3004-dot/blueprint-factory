# Art Direction: example-site

Date: 2026-06-26
Status: draft
Owner sign-off: pending

## 1. What this site is

An internal proof page for Blueprint Factory: a workshop that turns Codex into a repeatable high-craft website builder.

## 2. The visual world

Technical and precise, like a dark drafting table with glowing construction lines. It should feel intentional, not corporate.

## 3. The signature moment (required)

On load, the hero title slides through layered blueprint lines while a glowing cursor trace draws the first fold grid.

## 4. Motion language

- Pace: calm and deliberate.
- Easing feel: smooth and gliding.
- Scroll feel: light, with no heavy hijacking.
- Restraint: motion should frame the hero and not distract from the proof cards.

## 5. Typography

- Display type: large system sans with tight, architectural scale.
- Body type: system sans for fast local rendering.
- Character move: oversized title set against thin blueprint lines.

## 6. Color world

- Base: near-black drafting table.
- Accents: warm brass and cool blueprint blue.
- Overall feel: dark, crisp, and luminous.
- Contrast rule: accents should guide attention, not flood the page.

## 7. Layout system

- Grid feel: structured and architectural.
- Sections should differ through scale: first fold is expansive, proof section is dense and divided.
- First-screen plan: huge title, short explanation, and a visible drawing trace before scroll.

## 8. Reference comparanda

- Reference 1: Architectural blueprint drafting tables. Steal: thin construction lines and measured spacing.
- Reference 2: High-end studio landing pages. Steal: one strong first-screen gesture rather than many small effects.
- Reference 3: Motion identity reels. Steal: a single repeated trace that implies craft and process.

## 9. Anti-goals

- No centered gradient-blob hero.
- No generic SaaS card wall.
- No decorative motion that is disconnected from the blueprint idea.

## 10. Deploy and backend expectation

- Deploy profile expected: Vercel.
- Backend expected: none.

## 11. What must be true before this site is called ready

- Build passes.
- The signature trace is visible on desktop and mobile.
- Screenshots and a motion capture exist.
- Visual review records that this is an internal demo, not client-ready work.
